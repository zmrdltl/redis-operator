# Prometheus Monitoring

The redis-operator uses redis-exporter (opens new window) to expose metrics of redis setup in Prometheus format. This exporter captures metrics for both redis standalone and cluster setup.

This can be enabled and disabled from configuration. For example:-

redisExporter:
  enabled: true
  image: quay.io/opstree/redis-exporter:1.0
  imagePullPolicy: Always
  resources:
    requests:
      cpu: 100m
      memory: 128Mi
    limits:
      cpu: 100m
      memory: 128Mi

Once the exporter is configured, we may have to update Prometheus to monitor this endpoint. For Prometheus Operator (opens new window), we have to create a CRD based object called ServiceMonitor.

---
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: redis-monitoring-master
  labels:
    redis-operator: "true"
    env: production
spec:
  selector:
    matchLabels:
      role: master
  endpoints:
  - port: redis-exporter

---
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: redis-monitoring-slave
  labels:
    redis-operator: "true"
    env: production
spec:
  selector:
    matchLabels:
      role: slave
  endpoints:
  - port: redis-exporter
